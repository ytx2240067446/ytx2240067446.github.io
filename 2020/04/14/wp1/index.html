RIP

  • 拿 IDA 看 s 的 buf 大小为 0xF，故需要覆盖到返回地址为 0xF+8，即 23
  • 找到 system(‘/bin/sh’) 的地址为 0x000000000040118A
    • 有个问题就是不能直接用 fun 函数，原因不太明白…

warmup_csaw_2016

  • 找到 system(“cat flag.txt”) 的地址为 0x000000000040060d

  • s 的 buf 大小为 0x40

pwn1_sctf_2016

  • 找到 system(‘/bin/sh’) 的地址 0x08048F13
    • 不能直接调用 get_flag 函数
  • s 的 buf 大小为 0x3c，但是输入的字符串到不了这个长度
  • 逆向程序，会发现其会将 I 转换成 you ，利用此构造 0x3c / 3 长度的 I 就好了

ciscn_2019_n_1

  • 需要让 v2 = 11.28125
    • 也就是 0x41348000
  • 构造 payload 把它覆盖了就是了

ciscn_2019_c_1

  • 第一次自己写 ROP
  • 一直不成功，后来发现在 Ubuntu 18 上需要栈平衡
  • 输入的字符串会被加密，加密的过程是异或的，所以直接输入异或后的字符串可以变回来
    • 还可以在输入的字符串最前面输入 \0 骗程序字符串结束了

[OGeek2019]babyrop

  • 字符串比较可以在最前面输入 \x00 来欺骗此函数字符串长度为 0 进行绕过
  • 然后把 return 的值覆盖成 256 就能在后面的 read 函数里 ROP 了

get_started_3dsctf_2016

  • 这题直接跳到 get_flag 函数里 8 星，原因不明…

  • 只能用 mprotect 函数修改 bss 段执行权限后 ret2shellcode

[第五空间2019 决赛]PWN5

  • 上来看到一个格式化字符串漏洞，在读取名字输出的地方
  • 使用 aaaa.%08x.%08x.%08x.%08x.%08x.%08x.%08x.%08x.%08x.%08x 让其泄漏，发现在第 10 块内存地址
  • 于是构造 payload

[BJDCTF 2nd]r2t3

  • 刚开始以为要用 \x00 绕过 strlen 的检测，后来发现这玩意把 strcpy 也给绕了
  • 后来看到 name_check 函数里的 v3 只有 unsigned char 的大小，可以溢出
    • 那么超过 0xff 就会溢出，变成 0
  • 于是让 payload 长度到 260 就可以变成 4 了

ciscn_2019_n_8

  • 控制 var 数组第 13 位为 17 就好了，一个 int 的大小是 4，非常容易构造
