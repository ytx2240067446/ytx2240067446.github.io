  • 学校机房老搞笑了，交换机坏了彭哥说找机房总负责人，机房总负责人说网络办会修
  • 网络办的说他们不会修，要找机房总负责人，机房总负责人也说不会修
  • 不过最后自己鼓捣鼓捣就能用了
  • 网络办的人后来来了，帮忙测了机房的网口
  • 最后主路由搭配两个废旧的路由器弄了三个百兆的 Wifi
  • 加上交换机的有线网，老爽了

  • 网信办还整了个活，让每个机房的 IP 都在一个网段上混用

  • 而且学校的 DNS 服务器会把私自设置静态 IP 的用户全部禁止登陆

  • 由于机房有一台 NAS，如果没有固定的 IP，平时需要访问的时候就会很麻烦

  • 之前想到的解决方案是用主路由的 DDNS 将内网 IP 与域名绑定

  • 但是经测试发现 merlin 固件在 DDNS 解析的时候需要保证分配到的 IP 和 curl 的 IP 是一样的

    • 也就是说要有公网 IP 才能使用路由器自带的 DDNS，否则会解析失败

  • 由于固件是改版 merlin，在软件中心能找到很多第三方开发的 DDNS 插件

  • 但是无论是针对阿里云的 aliddns 插件还是针对 dnspod 的 ddnspod 插件都不能实现内网 IP 的 DDNS

    • 其默认都是外网 IP 的解析

  • 后来了解到似乎挺多域名服务商都不允许使用 DDNS 解析内网 IP

  • 经过查找，发现 dnspod 给的 API 是直接通过覆盖记录保存的值来修改 A 记录

    • 这样的话，直接修改 dnspod 的 DDNS 插件，让其得到的是内网 IP，就可以实现内网 IP 的 DDNS 了

  • 后来在研究插件的的时候发现 ddnspod 插件的中有一个函数没有被使用

    • 这个函数刚好就是用于获取路由器分配到的 IP 的，函数名叫作 arIpAdress
  • 直接修改插件把获取 IP 的函数改成 arIpAdress 就行

  • 注意配置 ddnspod 插件的时候，token 的形式是 ID,token
  • 而且现在又会发现一个问题，就是 ddnspod 插件没有办法正常获取结果
  • 经过搜索发现 dnspod 不能用 wget 函数来获取 API 的返回值，得用 curl
  • 这样的话修改插件，把原先的 wget 改成 curl 就行了

  • 内网 NAS 的 IP 就实现了 DDNS

  • 后来测试，使用学校提供的 webvpn 也可以访问到内网的 NAS，
  • 现在需要解决的就是在学校机房找个常年不断网断电的地方放一下 NAS
