覆盖栈上的内存

  • 格式化占位符中，%n 不会输出字符，但是会把前面已经成功输出的字符个数输入到对应位置的变量中

  • 设有如下程序

  • 构造字符串: [填充的内容] + [覆盖地址] + [填充的内容] + %[覆盖地址为第几个参数]$n
    • 覆盖地址前方的填充内容用于将覆盖的地址与机器字长对齐

  • 所以一般来说，也是如下步骤

    • 确定需要覆盖的地址
    • 确定相对偏移

  • 通过 gdb 调试，可以发现格式化字符串变量保存的地址在 0xffffd008，也就是第六个参数

  • 这样我们让 s 存储的值变为变量 c 的地址，然后使用 %n 往 c 写入数据
    • 假设我们需要写入的值为 16
    • 那么只需要在前面 printf 出十六个字符，然后用 %6$n 将 16 写入到变量 c 的地址中，那么 c 就改变了

覆盖大数字

  • 可以选择直接一次性输出大数字个字节来进行覆盖，但是这样基本也不会成功，因为太长了
  • 在x86和x64的体系结构中，变量的存储格式为以小端存储，即最低有效位存储在低地址
    • 0x12345678在内存中由低地址到高地址依次为\x78\x56\x34\x12

  • 格式化字符串的标识中，有两种特殊的标识

    • hh，将值先存为 char 类型，多余的删去，然后转换为 int
    • h， 将值先存为 short 类型，多余的删去，然后转换为 int

  • 因此可以利用 %hhn 向某个地址写入单字节，利用 %hn 向某个地址写入双字节
